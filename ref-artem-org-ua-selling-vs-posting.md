# @artem.org.ua — Selling vs Posting
*Ukrainian-language personal brand. Last updated: June 2026.*

---

## Default Mode: Professional

**Більшість постів — суто професійний контент. Ніяких особистих деталей.**

- Фокус: AI автоматизація для нетехнічного фахівця — конкретні кейси, механіки, числа.
- Без особистих historій (Бахмут, банкрутство, наркотики) — вони зарезервовані для рідкісних origin story постів, не для регулярного контенту.
- Голос — практик який робить, а не людина яка рефлексує.

Аудиторія — 5 сегментів нетехнічних українських фахівців. Вони хочуть бачити: ось конкретна проблема яку я впізнаю → ось як її вирішили без коду. Не мотивацію, не особисту drama.

---

## Головний принцип

**Пиши через досвід, не через продукт.**

Неправильно: "Я зробив інструмент для X — ось посилання."
Правильно: "Ось конкретна ситуація з мого життя де X коштував мені Y. Ось що я тепер роблю по-іншому."

Продукт — природний наслідок досвіду. Не навпаки.

## Структура контентної воронки

**80% — Професійний досвід і кейси**
Реальні ситуації з кар'єри і бізнес-процесів — конкретні задачі, AI-рішення, числа. Без продажу. Без посилань. Охоплює всі 5 ICP-сегментів. SC-кейси — не більше 20% від цього блоку.

**15% — Нішева експертиза**
AI-автоматизація, механіки, порівняння підходів — конкретні факти. Будує авторитет.

**5% — Прямий зв'язок з продуктом**
Конкретний пост про конкретний продукт або результат клієнта. З посиланням.

## Різниця між тим хто продає і тим хто ділиться

| Хто продає | Хто ділиться |
|---|---|
| Пише про продукт | Пише про проблему яку вирішує |
| Показує переваги | Показує власний провал і рішення |
| CTA в кожному пості | CTA рідко — і тому сильніше |
| Авторитет декларує | Авторитет демонструє через деталі |
| Говорить "вам потрібно" | Говорить "я так зробив" |

## Як вплітати продукт природньо

**Через особистий кейс:**
```
Я витратив 8 годин на аудит одного рахунку 3PL.
Знайшов $1,400 помилок.
Потім зробив інструмент який робить це за 15 хвилин.
Якщо потрібно — в біо.
```
(Ніяких прикметників. Ніяких обіцянок. Факт + факт + факт.)

**Через результат клієнта:**
```
Клієнт перевірив перший рахунок за допомогою [продукту].
$91,800 переплати знайдено за 8 місяців.
Не один рахунок — вся рахункова система.
```

**Через задачу яку аудиторія впізнає:**
```
Ти теж витрачаєш годину на пошук листа постачальника в Gmail?
Ось як я це вирішив — без коду, без нових сервісів.
Деталі в біо.
```

## Підхід "peer-to-peer", а не "вчитель до учня"

Артем — не коуч який навчає. Артем — людина яка пройшла через конкретне і ділиться механікою.

**Не так:** "Ключовий інсайт для вашого розвитку: завжди треба будувати системи."
**Так:** "Я не дисциплінований. Я системний. Ось різниця."

**Не так:** "Якщо ви хочете вийти на топ-рівень, потрібно..."
**Так:** "Я обрав новий напрямок кар'єри в кінці 2022. Менш ніж за 5 років — топ 1% зарплат. Ось що я робив конкретно."

## Розподіл особистого і професійного

**70% — Кар'єра, SC, AI, системи, рішення**
Це основа. Все що пов'язано з тим як Артем будував і перебудовував кар'єру + інструменти.

**30% — Особистість**
Деталі які роблять людину людиною. Читання як ресурс. Баклан (Shar-Pei). Шахи. Дубай. Бахмут. Ці деталі — не заповнювачі. Вони дають дозвіл аудиторії ідентифікуватися.

## Бекстейдж контент (1–2 рази на тиждень)

Не за сценарієм, без редагування:
- "Зараз будую X. Ось де застряг."
- "Тестую нову функцію. Ось перший результат."
- "Думаю над рішенням. Ось два варіанти — що б ти обрав?"

Бекстейдж дає відчуття реального процесу — не постановочного контенту.
