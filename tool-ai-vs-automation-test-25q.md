## How This Works

You have a specific task, process, or problem in mind.
Answer each question about that task on a scale from 0 to 4.

**Scoring scale:**
- **0** — Completely disagree / not true at all
- **1** — Slightly true
- **2** — Somewhat true
- **3** — Mostly true
- **4** — Completely true / always the case

**Higher score = task leans toward Automation**
**Lower score = task requires AI**

## Your Result

| Score | Verdict | What it means |
|-------|---------|---------------|
| **76–100** | Classic Automation | Pure rule-based task. Use workflow tools: Zapier, Make, Power Automate, Python scripts, RPA. No AI needed — it would only add cost and complexity. |
| **56–75** | Automation-First + Light AI | Core process is automatable, but some steps need AI for exceptions or unstructured inputs. Start with automation, add AI for the edge cases. |
| **36–55** | Intelligent Automation | You need both. AI handles the judgment, prediction, or language understanding — automation handles execution and routing. This is the most common real-world answer. |
| **0–35** | AI-First | This task is too variable, judgment-heavy, or adaptive for classic automation. You need machine learning, an LLM, or an AI agent. Automation alone will break constantly. |

## The Core Distinction (in plain English)

**Automation** follows rules you write.
It does exactly what you tell it — always.
It's fast, cheap, and reliable — as long as nothing unexpected happens.

**AI** learns from examples.
It handles situations you didn't explicitly program.
It understands text, detects patterns, makes predictions, and adapts.

**The question is not "which is better" — it's "which does this task actually need?"**

Using AI where automation would work = unnecessary cost.
Using automation where AI is needed = constant breakdowns and manual fixes.
