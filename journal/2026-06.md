## 2026-06-10/11 — FlowerOS architecture + implementation plan

**Topic:** FlowerOS — B2B Telegram loyalty bot для квіткових магазинів. Від ідеї до архітектурного spec + Plan 1.

**Context:**
Artem має реального клієнта (Дніпро, магазин з обігом 600K UAH/міс, середній чек 1,200 UAH). Комісія: 7% без setup fee. Ціль: 1,500 активних юзерів на магазин → ~47K UAH/міс. Scale target: 9 магазинів = $10K/міс.

**Що зробили (session 1 — архітектура):**
- Оновили документи (10% → 7% комісія, прибрали setup fee) у `plan-flower-loyalty-system-concept.md` та `plan-flower-loyalty-pitch.md`
- Прорахували unit economics для Дніпра з нуля: 500 тр/міс ÷ 2 покупки/рік = 3,000 унікальних клієнтів/рік, 50% реєстрація = 1,500 активних
- Спроектували повну архітектуру через Visual Companion (браузерний companion): actors+core, integrations+storage, flows
- Ключові архітектурні рішення:
  - Multi-tenant: один Python сервіс, окремий bot token на магазин
  - Airtable як оперативна БД (1 base per shop, 8 таблиць)
  - LMachine.io як loyalty backend (весь розрахунок кешбеку делегований)
  - Flora24 API: bidirectional (GET stock + POST pre-orders)
  - Monobank ekvayring + cash (+3% bonus cashback) + terminal
  - Profile sharing: вірусні deep links `t.me/bot?start=p_{profile_id}`, gender-neutral
  - AI Support Agent: claude-haiku-4-5, tools: get_stock / get_cashback_balance / escalate
  - APScheduler: щоденний scan о 09:00 + seasonal campaigns
- Зробили PDF-ready HTML: `ref-floweros-architecture.html` (standalone, landscape A4, break-inside: avoid)
- Написали та закомітили spec: `docs/superpowers/specs/2026-06-10-floweros-architecture-design.md`

**Що зробили (session 2 — Plan 1):**
- Розбили реалізацію на 3 плани: Core Bot (Plan 1), Integrations (Plan 2), Advanced (Plan 3)
- Написали повний Plan 1 (11 tasks, ~45 tests, TDD):
  - Task 1: Project setup
  - Task 2: Config loader (YAML → ShopConfig, env var resolution)
  - Task 3: Data models (User, Profile, ProfileDate, Order)
  - Task 4: AirtableClient (create/get для Users, Profiles, Orders)
  - Task 5: Bot states enum + InlineKeyboard builders
  - Task 6: Onboarding ConversationHandler (8-крок survey)
  - Task 7: Profile management (list + add flow)
  - Task 8: Scheduler (daily scan → pending_reminders dict)
  - Task 9: Order flow (budget → manager notify → delivery → payment method)
  - Task 10: Bot factory + main.py (multi-tenant polling)
  - Task 11: Manual smoke test
- Закомітили план: `docs/superpowers/plans/2026-06-10-floweros-bot.md`

**Key decisions:**
- Pending reminders tracked via `bot_data["pending_reminders"][tg_id] = profile.airtable_id`
- Order flow driven by global MessageHandler + CallbackQueryHandlers (не ConversationHandler) — спрощення для Plan 1
- dates зберігаються в Airtable як JSON string (не link fields)
- Manager photo forwarding — відкладено на Plan 2
