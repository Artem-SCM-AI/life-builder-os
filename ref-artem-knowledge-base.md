# PART 6: SUPPLIER COMMUNICATION AI — FULL DELIVERY SYSTEM

## 6.1 SCAI Workflow (Step-by-Step)

### What the service does

Artem receives forwarded supplier emails, extracts all operational data using Claude, drafts a professional reply, and logs everything in the client's shared Google Sheet — delivered within 4 business hours.

**What the client provides:**
- Forwarded supplier emails
- Supplier context filled during onboarding (names, SKUs, standard terms)

**What the client receives:**
- Structured data extract (lead time, price, MOQ, production status, ETD/ETA, flags)
- Ready-to-send reply draft (max 150 words, client's preferred tone)
- Updated Google Sheet log entry
- Immediate WhatsApp/email alert if any threshold is breached

### Step 1 — Receive and Log Inbound Email

1. Open forwarded email in Gmail
2. Copy full email body (including headers: From, Date, Subject)
3. Open Google Sheets → SCAI — [ClientName] → Email Log tab
4. Add new row: Date, Supplier Name, Subject, Raw Email, Status = "Processing"

### Step 2 — Run Claude Parsing Prompt

Paste into Claude:

```
You are a supply chain analyst processing a supplier email for an Amazon FBA brand. Extract all operational data and draft a professional reply.

SUPPLIER EMAIL:
[PASTE FULL EMAIL HERE — including From, Date, Subject, Body]

SUPPLIER CONTEXT:
- Supplier name: [NAME]
- Product / SKU: [PRODUCT/SKU]
- Relationship length: [X months / X years]
- Last order: [DATE, QTY, VALUE if known]
- Client reply tone preference: [FORMAL / FRIENDLY]

OUTPUT FORMAT:

## Extracted Data
| Field | Value | Change from Last |
|-------|-------|-----------------|
| Lead Time | | |
| Unit Price | | |
| MOQ | | |
| Production Status | | |
| ETD (Factory) | | |
| ETA (Port / Warehouse) | | |
| Payment Terms | | |
| Issues / Flags | | |

## Key Alerts
List any of the following if present: price change >5%, lead time shift >7 days, quality issue mentioned, supplier threatens delay or cancellation. If none: write "None."

## Suggested Reply
[Professional email reply addressing all points raised. Tone: FORMAL or FRIENDLY as specified. Max 150 words.]

## Internal Note
[1–2 sentences: what action does the client need to take, if any?]
```

### Step 3 — Review Claude Output

- Check the Extracted Data table for completeness
- Check Key Alerts — if any exist, contact client immediately via WhatsApp before delivering
- Review Suggested Reply: correct tone, accurate, under 150 words

### Step 4 — Update Google Sheets Log

Fill columns: Lead Time, Unit Price, MOQ, Production Status, ETD, ETA, Payment Terms, Flags, Suggested Reply, Status (Processed or ALERT — Review Needed)

### Step 5 — Deliver to Client

Email/WhatsApp subject: `SCAI Reply Ready — [Supplier Name] — [YYYY-MM-DD]`

### Step 6 — Close the Loop

- Add Delivered At timestamp
- Update Status to Delivered → then Closed after client confirms
- Monthly 15-min review call with client

## Post 4 — How-To / Tip (May 1)

How I process a supplier email in under 5 minutes:

Step 1: Paste the full email into Claude with this prompt:
"Extract: price, lead time, MOQ, production status, ETD, ETA, payment terms. Flag any changes from last known values."

Step 2: Review the extracted table (30 seconds).

Step 3: Check alerts. Price up >5%? Lead time shifted >7 days? Flag the client immediately.

Step 4: Review the reply draft. Edit if needed — usually 1–2 lines max. Send.

Step 5: Log extracted data in the supplier sheet. One row. All fields. Done.

# PART 11: AMAZON FBA OPERATIONS REFERENCE

## 11.1 FBA Inbound Receiving Time (Arrival → Available in Stock)

*Researched May 29, 2026 | Sources: SupplyKick, Freightos, Forceget, Seller Central Forums, TikTok @mrbruks, Instagram @tacticallogistic*

### Статуси відвантаження (по порядку)

```
Вантаж приїхав на FC → Checked In → Receiving → Available (перші юніти)
                                                → Fully Distributed (вся мережа)
```

- **Checked In** — перший ярлик відсканований, трейлер у дворі фасіліті
- **Receiving** — активна обробка одиниць (розпакування, скан, розміщення)
- **Available** — перші юніти можна продавати (з'являються під час receiving, поступово)
- **Available for immediate shipping** — повний розподіл по FC-мережі

### Час по типу відвантаження

| Тип відвантаження | Checked In → Receiving | Перші юніти в стоку | Реальний досвід продавців |
|-------------------|------------------------|---------------------|---------------------------|
| **Small Parcel (SPD)** | до 2 днів | 3–5 робочих днів | 10–14 днів |
| **LTL / FTL (палети)** | до 9 днів | 10–14 робочих днів | 3–4 тижні |
| **Q4 / Prime Day (пік)** | до 2 тижнів | до 30 днів | 24+ днів (типово) |

**Офіційна позиція Amazon:** до 30 днів від доктора до повного стоку — зовнішня межа; більшість проходить швидше.

**Ключова деталь:** Перші юніти стають available **одразу під час receiving** — shipment ще обробляється, а частина товару вже в стоку. Для **повного розподілу по мережі** FC — ще до 18 днів після receiving.

### Що впливає на швидкість (2026)

**Прискорює:**
- 2D-штрихкоди (GS1-128 QR) — скорочують check-in на **2 тижні** (підтверджено практикою)
- Точна відповідність між shipment plan і фізичним вантажем (box content, dimensions, weights)
- Правильне маркування з першого разу (FNSKU на кожному юніті)

**Уповільнює:**
- З 1 січня 2026: Amazon більше **не надає prep і labeling services** для US-відвантажень → будь-яка помилка маркування = затримка
- Перевантаженість конкретного FC
- Drop trailer без appointment
- Невідповідність carton dimensions / quantities у shipment plan
- Неправильний carrier (третій перевізник без знання Amazon вимог)

### Практичний ориєнтир для планування lead time

| Сценарій | Додай до lead time |
|----------|--------------------|
| SPD, нормальний сезон | +7–10 днів від доставки |
| LTL/FTL, нормальний сезон | +14–21 день від доставки |
| Будь-який тип, Q4 / Prime Day | +4–5 тижнів від доставки |
| Найгірший сценарій (помилки + пік) | до 30+ днів |

**Правило від форумів:** Бюджетуй 1 місяць для SPD і 2 місяці для FTL/LTL щоб inventory стало available, якщо хочеш бути в безпеці.
