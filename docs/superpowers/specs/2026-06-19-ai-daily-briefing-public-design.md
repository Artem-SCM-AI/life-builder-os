# AI Daily Briefing — Public GitHub Repo Design

**Date:** 2026-06-19
**Source:** tool-morning-briefing/ (existing, 35 tests)
**Scope:** Refactor into standalone public product with adaptive setup wizard and modular task manager adapters

---

## Overview

A macOS CLI tool that sends three automated briefings via Telegram daily. Claude analyzes tasks and personal focus, then asks accountability questions. Each briefing feeds the next — closed loop.

Target audience: macOS users with a Telegram bot, optionally using ClickUp or Notion.

## TaskAdapter Protocol (adapters/base.py)

```python
from dataclasses import dataclass
from typing import Protocol

@dataclass
class Task:
    name: str
    priority: str = "normal"  # "high" | "normal" | "low" — Notion returns "normal" always
    due: str | None = None    # "2026-06-19" or None

class TaskAdapter(Protocol):
    def get_tasks_due_today(self) -> list[Task]: ...
    def get_tasks_closed_today(self) -> list[Task]: ...
    def get_tasks_created_today(self) -> list[Task]: ...
    def get_tasks_closed_this_week(self) -> list[Task]: ...
    def get_tasks_created_this_week(self) -> list[Task]: ...
```

### adapters/clickup.py
Renamed from `clickup_client.py`. Wraps existing functions into a class implementing `TaskAdapter`. No logic changes.

### adapters/notion.py
Queries a Notion database. Requires: `NOTION_TOKEN`, `NOTION_DATABASE_ID`, `NOTION_DONE_PROPERTY` (default: `Status`), `NOTION_DONE_VALUE` (default: `Done`).

- `get_tasks_due_today`: filter `Due` date property = today AND status != done
- `get_tasks_closed_today`: filter last_edited_time = today AND status = done
- `get_tasks_created_today`: filter created_time = today
- `get_tasks_closed_this_week` / `get_tasks_created_this_week`: same logic, Mon–Fri range

### adapters/none.py
Returns `[]` for all methods. Used when `TASK_MANAGER=none`.

### Adapter factory (briefing.py)

```python
def load_adapter(task_manager: str) -> TaskAdapter:
    if task_manager == "clickup":
        return ClickUpAdapter(
            token=os.environ["CLICKUP_TOKEN"],
            team_id=os.environ["CLICKUP_TEAM_ID"],
        )
    if task_manager == "notion":
        return NotionAdapter(
            token=os.environ["NOTION_TOKEN"],
            database_id=os.environ["NOTION_DATABASE_ID"],
            done_property=os.environ.get("NOTION_DONE_PROPERTY", "Status"),
            done_value=os.environ.get("NOTION_DONE_VALUE", "Done"),
        )
    return NoneAdapter()
```

## reader.py — Simplified

Removes multi-file context reading. Reads `focus.md` directly as a string.

```python
def read_focus(path: str) -> str:
    try:
        return Path(path).read_text(encoding="utf-8")
    except FileNotFoundError:
        return "[Focus file not found. Run setup.py to create it.]"
```

`extract_current_focus()` removed — focus.md is the full context, no section extraction needed.

## briefing.py — Key Changes

1. `_claude_path() -> str`: `shutil.which("claude") or os.path.expanduser("~/.local/bin/claude")` — replaces hardcoded `CLAUDE_PATH` constant. Used by both `generate_briefing()` and `update_focus()`.
2. `run_morning()`: replaces 3-file reader with `read_focus(os.environ["FOCUS_MD_PATH"])`
3. All ClickUp direct imports replaced by `adapter = load_adapter(os.environ["TASK_MANAGER"])`
4. `run_weekly()`: after sending Telegram message, calls `update_focus(adapter, today)` to regenerate focus.md

```python
def update_focus(adapter: TaskAdapter, today: date) -> None:
    closed = adapter.get_tasks_closed_this_week()
    current_focus = read_focus(os.environ["FOCUS_MD_PATH"])
    prompt = (
        "Based on this week's completed tasks and the current focus, "
        "write an updated focus.md for next week. "
        "Plain text only. No markdown. No headers. 150 words max.\n\n"
        f"Current focus:\n{current_focus}\n\n"
        f"Completed this week: {[t.name for t in closed]}"
    )
    # Direct subprocess call — bypasses SYSTEM_PROMPT (numbered-answers format not suitable here)
    result = subprocess.run(
        [_claude_path(), "--print", prompt],
        capture_output=True, text=True, timeout=60,
    )
    if result.returncode == 0:
        Path(os.environ["FOCUS_MD_PATH"]).write_text(result.stdout.strip(), encoding="utf-8")
```

## Setup Wizard (setup.py)

Two blocks. Runs once. Requires Claude CLI installed.

### Block A — Technical config

```
Which task manager do you use?
  1. ClickUp
  2. Notion
  3. None

[ClickUp]
  ClickUp API token: pk_...
  ClickUp Team ID: ...

[Notion]
  Notion API token: secret_...
  Notion database URL or ID: ...
  Property name for status (default: Status):
  Value that means "done" (default: Done):

Telegram bot token: ...
Telegram chat ID: ...
Timezone (default: Europe/Kiev):
Morning briefing time (default: 08:45):
Evening briefing time Mon-Thu (default: 20:45):
Evening briefing time Fri (default: 20:00):
Weekly briefing time Fri (default: 20:05):
```

### Block B — Focus interview (4 questions)

4 questions via `input()`:

```
1. What's your role — what do you do?
2. What are your top 2-3 goals for this quarter?
3. What's your biggest current blocker or challenge?
4. What does success look like for you in 3 months?
```

Answers passed to Claude:
```
claude --print "You are creating a personal focus file. Write focus.md based on these answers: [answers]. Plain text. No markdown headers. 150 words max."
```

Output written to `focus.md`.

### Block C — Finalize

1. Write `config.env`
2. Copy plist templates → `~/Library/LaunchAgents/` substituting `{{USERNAME}}`, `{{REPO_PATH}}`, `{{MORNING_TIME}}`, `{{EVENING_TIME}}`, `{{EVENING_FRI_TIME}}`, `{{WEEKLY_TIME}}`
3. `launchctl load` each plist
4. `send_telegram(token, chat_id, f"Briefing system ready. First morning briefing at {morning_time}.")` — uses the time entered in Block A

## LaunchAgent Templates

Plist templates use `{{USERNAME}}` and `{{REPO_PATH}}` placeholders. Setup substitutes them with `getpass.getuser()` and `Path(__file__).parent.resolve()`.

Label format: `com.{{USERNAME}}.morning-briefing` (not hardcoded to "artem").

Each plist passes these `EnvironmentVariables` keys (sourced from config.env at substitution time):

```
HOME, PATH, TASK_MANAGER, TELEGRAM_TOKEN, TELEGRAM_CHAT_ID,
FOCUS_MD_PATH, TZ,
CLICKUP_TOKEN, CLICKUP_TEAM_ID,           (only if TASK_MANAGER=clickup)
NOTION_TOKEN, NOTION_DATABASE_ID,          (only if TASK_MANAGER=notion)
NOTION_DONE_PROPERTY, NOTION_DONE_VALUE    (only if TASK_MANAGER=notion)
```

Unused keys are omitted from the plist (setup.py writes only relevant keys based on chosen task manager).

## Weekly Focus Update

Runs at end of `run_weekly()`. No user input required mid-flow.

Claude receives: current `focus.md` + list of tasks closed this week + count of created tasks.
Claude outputs: new `focus.md` content (plain text, ~150 words).
System writes to `FOCUS_MD_PATH` silently.

User sees the updated focus reflected in the next Monday morning briefing.
