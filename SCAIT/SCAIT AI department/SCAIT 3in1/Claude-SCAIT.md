# Claude-SCAIT.md

> Інструкції для Claude Code при роботі в робочому просторі маркетингової команди SCAIT 3in1.

---

## Про цей workspace

Це робочий простір маркетингової команди SCAIT (Supply Chain AI Tools).  
Тут зберігаються контекст бренду, процеси, шаблони та готові результати по трьох підрозділах.

```
SCAIT 3in1/
├── Marketing/
│   ├── _context/       — базові файли контексту бренду (джерело правди)
│   ├── _sop/           — стандартні операційні процедури маркетингових процесів
│   ├── _templates/     — шаблони для повторного використання
│   ├── _social/        — готовий контент для соцмереж
│   ├── _ads/           — рекламні матеріали
│   ├── _pages/         — копірайт і вайрфрейми лендингів
│   ├── _seo/           — SEO-дослідження, брифи, метадані
│   ├── _research/      — дослідження ринку, ICP, конкуренти
│   ├── _reports/       — звіти та аналітика
│   └── _presentations/ — презентації та PDF-матеріали
├── Sales/              — аутріч, скрипти, пропозиції, CRM
└── Ops/                — процеси доставки, онбординг клієнтів
```

---

## Файли контексту бренду

Всі знаходяться в `Marketing/_context/`. Підгружай лише ті, що безпосередньо відносяться до задачі.

| Файл | Завантажити коли |
|------|-----------------|
| `SCAIT-BRAND-BOOK.md` | Будь-який зовнішній output: дизайн, позиціонування, структура матеріалів |
| `SCAIT-TONE-OF-VOICE.md` | Написання будь-якого тексту: пост, email, скрипт, лендінг |
| `SCAIT-PRODUCTS_OFFERING.md` | Продуктові описи, продажні матеріали, ROI-аргументи |
| `SCAIT-GROWTH-MARKETING-CONTEXT.md` | Стратегія, канали, ICP, контент-план, аутріч |

## Правила генерації

### Всі тексти і контент

Всі генерації повинні слідувати гайдам з голосу бренду з `_context/`. Перед початком роботи над зовнішнім контентом підгружати мінімум `SCAIT-TONE-OF-VOICE.md`.

### Skills

- **Незалежні від бренду** — визначають лише workflow і процес.
- Жодних hardcoded деталей бренду: назв продуктів, цін, ROI-цифр, ICP-описів.
- Бренд-специфічний контекст підається через промпт або context-файли під час виклику.

### Агенти

- **Незалежні від бренду** — логіка агента не знає, що він працює для SCAIT.
- Кожен агент має чітку, непересічну роль.
- Бренд-контекст передається агенту через system prompt або аргументи при виклику.

## Де зберігати результати

| Тип результату | Папка |
|---------------|-------|
| Пости, тред, репости | `Marketing/_social/` |
| Рекламні креативи та копі | `Marketing/_ads/` |
| Копірайт лендингів | `Marketing/_pages/` |
| SEO-брифи, ключові слова, метадані | `Marketing/_seo/` |
| Дослідження ринку та ICP | `Marketing/_research/` |
| Аналітичні звіти | `Marketing/_reports/` |
| Презентації, decks | `Marketing/_presentations/` |
| Нові SOP | `Marketing/_sop/` |
| Нові шаблони | `Marketing/_templates/` |

## Формат відповідей

- Deliverable — одразу без преамбули.
- Стратегічне питання — таблиця варіантів з трейдофами + рекомендація.
- Правки — вносити напряму, не пояснювати що змінилось якщо не питають.
- Мова комунікації: українська. Мова deliverables: англійська (якщо не вказано інше).
